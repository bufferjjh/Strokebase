S T R O K E B A S E

Overview

Strokebase was built to provide a quick and precise prediction of one’s stroke risk using machine learning. After answering a series of questions, users can see the model’s prediction along with the prediction confidence. For each question, Strokebase cites a credible source that describes its relation to stroke. However, given the brief and surface-level nature of Strokebase, users should NOT use predictions as a real medical diagnosis. Strokebase is unable to perform physical tests and does not cover a variety of other factors that may be related to stroke. To receive a comprehensive assessment, users should always visit a medical professional.

Development

The heart of Strokebase is its prediction model. The model was built using Python since the language allowed the use of many useful libraries including Sklearn, NumPy, and Pandas. Pandas was used to clean and represent the data in a DataFrame. Since there was incomplete and non-numerical data, Pandas allowed for easy cleaning and encoding. Strokebase’s model also made use of the Logistic Regression Model which is part of SkLearn. Logistic Regression is commonly used to predict the probability of an event happening, which is stroke or no stroke in this case. After training the model, Python’s pickle module allowed the model object to be saved as a file. Therefore, the model didn’t have to be trained repeatedly and models with high accuracy could be saved. Over 100 iterations of training, the best model was saved with an accuracy score of 95.63% (Rounded to the nearest hundredth)

The backend server was programmed using Flask. Since the backend server also used Python, it could easily access prediction functions from the model (also built using python). Personally, I think Flask syntax is pretty clear with requests and responses. The front-end was built using React. This made rendering different questions to the page easier as each question could be treated as a component. JSX also made creating components easier since HTML could be written within JavaScript.
